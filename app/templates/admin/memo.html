Must Read Before Starting A Session

1. For UNPACED Session with SAMPLE POOL:

- Complete (in order) Parameter - Files - Demand tabs before the start of class;

- Content in Demand tab should be automatically generated without error messages (validation);


2. For UNPACED Session with DISTRIBUTION (Normal, Uniform, Triangular):

- Complete (in order) Parameter - Demand tabs before the start of class;

- Content in Demand tab should be automatically generated without error messages (validation);


3. For PACED Session with SAMPLE POOL or DISTRIBUTION (Normal, Uniform, Triangular):

- Complete step 1 or step 2 depends on demand pattern before the start of class;

- Complete Pace tab (after all students logged in with session code) to start the session at the beginning of class;

- Content in Demand tab and Pace tab should be automatically generated without error messages (validation);


Parameter Tab Creation Format

1. Parameter ID (par_id):

- Given as primary key by default;


2. Session Code (scode):

- For Harish's class, 'harishk_XX' (all lower cases), XX represents 2 integers (e.g. 01) to uniquely identify different class/game sessions;

- For Tim's class, 'timh_XX' (all lower cases), XX represents 2 integers (e.g. 01) to uniquely identify different class/game sessions;


3. Indicator of Consistent Pace Game (consistent_pace):

- Checkbox;

- Check indicates it's a consistent pace game;

- Uncheck indicates it's a regular (unpaced) game;

- In paced games, all students move along with each round at the same time, and therefore ranks of round profit & total profit will be provided to each student at the end of each round;


4. Number of Rounds (nrounds):

- Integer;

- >=10, <= 50;


5. Wholesale Price (wholesale_price):

- Float, round to 2 decimal places;

- >= 0;


6. Retail Price (retail_price):

- Float, round to 2 decimal places;

- > 0, > whole_price;


7. Demand Pattern (demand_pattern):

- 1 of 4 options;

- Sample pool (past and new demands will be summarized in a sample pool in order and given as a txt file);

- Normal distribution;

- Triangular distribution;

- Uniform distribution;


8. Sample Pool Input File Name (sample_pool_input_file_name_txt):

- A string of input file name;

- Input file must be in .txt format;

- Input file must only contain integer demand amount (seperated by comma);

- Number of integer in input file must be greater or equal to number of past demand show plus number of rounds;


9. Number Of Past Demand Show As Reference For Users To Make Decision (sample_pool_number_of_past_demand_show):

- Integer;

- >= 0;

- First N integers from sample pool input file will be shown to users as historical demand reference when making decisions;


10. Demand Lower Bound (distribution_all_lower_bound):

- Integer;

- >= 0;

- Not null if demand pattern is from distribution (not from sample pool);


11. Demand Upper Bound (distribution_all_upper_bound):

- Integer;

- > 0, > demand_lower_bound;

- Not null if demand pattern is from distribution (not from sample pool);


12. Mean For Normal Distribution (distribution_normal_mean):

- Integer or float;

- > 0;

- Not null if demand pattern is from normal distribution;


13. Standard Deviation For Normal Distribution (distribution_normal_std):

- Integer or float;

- > 0;

- Not null if demand pattern is from normal distribution;


14. Peak For Triangular Distribution (distribution_triangular_peak):

- Integer or float;

- > 0;

- Not null if demand pattern is from triangular distribution;


Demand Tab Creation Format

1. Must be created after creating parameters;

2. Upload corresponding sample pool input file (stated in Parameter tab) in File tab before making any changes in Demand tab;

3. By default (if everything is valid), session code, past demands, and new demands should appear in input boxes automatically;

4. If there are errors, refer to error message in input box and make changes correspondingly;


Pace Tab Creation Format

1. Must be created after creating the demand tab (without errors);

2. Must be created after all students logged in with the session code at the beginning of the class (check if session size equals true class size);

3. By creating the Pace tab, the paced game session will be started, and new or late students won't be able to log into the session afterwards even with correct session code;

4. By default (if everything is valid), session code, session size, and session ID collection should appear in input boxes automatically;

5. If there are errors, refer to error message in input box and make changes correspondingly;
